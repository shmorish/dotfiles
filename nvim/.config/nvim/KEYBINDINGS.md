# Neovim キーバインド・設定ガイド

## Vimの基本モード

Vimには複数のモードがあります。現在のモードは画面左下に表示されます。

| モード | 説明 | 入り方 |
|--------|------|--------|
| **Normal** | 移動・操作の基本モード | `<ESC>` または `jj` |
| **Insert** | テキスト入力モード | `i`（カーソル前）、`a`（カーソル後）、`o`（次の行）など |
| **Visual** | テキスト選択モード | `v`（文字選択）、`V`（行選択）、`<C-v>`（矩形選択） |
| **Command** | コマンド実行モード | `:` |
| **Terminal** | 組み込みターミナル | `<leader>tt` など |

---

## 1. 基本設定

| 設定 | 値 | 説明 |
|------|----|------|
| **リーダーキー** | `Space` | `<leader>` はスペースキー |
| **タブ** | スペース4つ | `expandtab` で実際のタブ文字は使用しない |
| **行番号** | 絶対行番号 + 相対行番号 | 相対番号は `rn` でトグル可能 |
| **undofile** | 有効 | ファイルを閉じても Undo 履歴が保持される |
| **クリップボード** | システム共有 | ヤンク/ペーストがシステムクリップボードと連動 |
| **検索** | 大文字小文字スマート判定 | 小文字で検索すると大文字小文字を無視 |
| **スクロールオフセット** | 4行 | カーソル上下に常に4行の余白を確保 |

## 2. 基本キーバインド

### モード変更

| キー | モード | 動作 |
|------|--------|------|
| `jj` | Insert | Normal モードへ戻る |

### 表示

| キー | モード | 動作 |
|------|--------|------|
| `rn` | Normal | 相対行番号のオン/オフ切り替え |
| `<ESC><ESC>` | Normal | 検索ハイライトを消去 |
| `<leader>n` | Normal | 検索ハイライトを消去 |

### カーソル移動

| キー | モード | 動作 |
|------|--------|------|
| `j` | Normal/Visual | 折り返し行を考慮して下に移動（`gj`） |
| `k` | Normal/Visual | 折り返し行を考慮して上に移動（`gk`） |
| `H` | Normal/Visual | 行頭へ移動（`0`） |
| `L` | Normal/Visual | 行末へ移動（`$`） |

### ウィンドウ操作

| キー | 動作 |
|------|------|
| `sh` / `sj` / `sk` / `sl` | 左/下/上/右のウィンドウへ移動 |
| `sH` / `sJ` / `sK` / `sL` | ウィンドウを左/下/上/右に移動（強制） |
| `ss` | 水平分割（上下） |
| `sv` | 垂直分割（左右） |
| `s,` | ウィンドウ幅を縮める（-3） |
| `s.` | ウィンドウ幅を広げる（+3） |
| `s-` | ウィンドウ高さを縮める（-3） |
| `s=` | ウィンドウ高さを広げる（+3） |
| `s0` | 全ウィンドウを均等サイズに戻す |

> `s,` や `s-` などのリサイズは `.` で繰り返し実行できます（vim-repeat）

### 編集

| キー | 動作 |
|------|------|
| `+` | カーソル下の数値を +1 |
| `-` | カーソル下の数値を -1 |
| `Y` | 行末までヤンク（`y$`） |
| `p` | インデントを調整してペースト（`]p`） |
| `P` | インデントを調整して前にペースト（`]P`） |

### ターミナル

| キー | 動作 |
|------|------|
| `<leader>tt` | 現在のウィンドウでターミナルを開く |
| `<leader>ts` | 下に分割してターミナルを開く（高さ15行） |
| `<leader>tv` | 右に分割してターミナルを開く |
| `<ESC>` | ターミナルモード終了（Normal モードへ） |

### コマンド（大文字でも動作）

| キー | 動作 |
|------|------|
| `Q` | `:q`（ウィンドウを閉じる） |
| `W` | `:w`（ファイルを保存） |
| `QA` | `:qa`（全ウィンドウを閉じる） |
| `WQ` | `:wq`（保存して閉じる） |
| `WQA` | `:wqa`（全ウィンドウを保存して閉じる） |

## 3. プラグイン別キーバインド

### Telescope（ファイル・テキスト検索）

ファジーファインダー。素早くファイルや文字列を検索できます。

| キー | 動作 |
|------|------|
| `<leader>ff` | ファイル名で検索 |
| `<leader>fg` | プロジェクト全体をgrep検索（ライブ） |
| `<leader>fb` | 開いているバッファ一覧 |
| `<leader>fh` | Vimのヘルプを検索 |

### Fern（ファイルツリー）

左右に表示されるファイルエクスプローラー。

| キー | 動作 |
|------|------|
| `<leader>e` | ファイルツリーのトグル（右側ドロワー） |
| `l` | ディレクトリを展開 / ファイルを開く |

> ファイルツリー内では `s` キーはウィンドウ操作と衝突するため無効化されています。

### mini.nvim

| キー | 機能 | 動作 |
|------|------|------|
| `<leader>l` | mini.bracketed | 次のバッファへ移動 |
| `<leader>h` | mini.bracketed | 前のバッファへ移動 |
| `gc` + 移動 | mini.comment | コメントのトグル（例: `gcc` で1行、`gcip` で段落） |
| `sa` + 移動 | mini.surround | 囲み文字を追加（例: `saiw"` で単語を `"` で囲む） |
| `sd` | mini.surround | 囲み文字を削除（例: `sd"` で `"` を削除） |
| `sr` | mini.surround | 囲み文字を置換（例: `sr"'` で `"` を `'` に変更） |

> その他 mini.pairs（括弧の自動補完）、mini.cursorword（カーソル下の単語をハイライト）なども有効です。

### Hop（高速カーソル移動）

画面内の任意の位置にジャンプできます。

| キー | 動作 |
|------|------|
| `f` + 文字 | カーソル後方の1文字ヒントで移動（同一行） |
| `F` + 文字 | カーソル前方の1文字ヒントで移動（同一行） |
| `t` + 文字 | `f` と同様だがターゲットの1文字手前に止まる |
| `T` + 文字 | `F` と同様だがターゲットの1文字後ろに止まる |

> ヒント文字が表示されたら、移動先のラベル文字を入力するとジャンプします。

### Trouble（診断・エラー一覧）

LSPの診断結果やエラーを一覧表示するパネルです。

| キー | 動作 |
|------|------|
| `<leader>xx` | 全診断一覧のトグル |
| `<leader>xX` | 現在のバッファの診断一覧のトグル |
| `<leader>cs` | シンボル一覧（関数・変数などの構造） |
| `<leader>cl` | LSP定義/参照一覧（右側パネル） |
| `<leader>xL` | Location List のトグル |
| `<leader>xQ` | Quickfix List のトグル |

### Persistence（セッション管理）

プロジェクトごとに開いていたファイルやウィンドウ配置を保存・復元します。

| キー | 動作 |
|------|------|
| `<leader>qs` | 現在のディレクトリのセッションを復元 |
| `<leader>qS` | セッションを選択して復元 |
| `<leader>ql` | 最後のセッションを復元 |
| `<leader>qd` | セッションの自動保存を停止 |

### nvim-cmp（補完）

コード補完のキーバインドです（Insert モード）。

| キー | 動作 |
|------|------|
| `<C-Space>` | 補完メニューを手動で開く |
| `<CR>` | 補完候補を確定 |
| `<C-e>` | 補完をキャンセル |
| `<C-b>` | ドキュメントウィンドウを上にスクロール |
| `<C-f>` | ドキュメントウィンドウを下にスクロール |
| `<Tab>` / `<S-Tab>` | 補完候補の選択（上下） |

## 4. LSP キーバインド

LSP（Language Server Protocol）が有効なバッファで使えるキーバインドです。
コードの定義参照・補完・リファクタリングなどを行います。

| キー | 動作 |
|------|------|
| `K` | ホバー（カーソル下のドキュメント表示） |
| `gr` | 参照一覧を表示 |
| `gd` | 定義へジャンプ |
| `gD` | 宣言へジャンプ |
| `gi` | 実装へジャンプ |
| `ge` | 診断メッセージをフロート表示 |
| `g]` | 次の診断（エラー/警告）へ移動 |
| `g[` | 前の診断（エラー/警告）へ移動 |
| `gf` | ファイルをフォーマット |
| `gn` | シンボルをリネーム |
| `ga` | コードアクションを表示 |

> 診断はエラー（Error）と警告（Warning）以上が下線表示されます。
